Project

Design two task flows, an admin and a basic user role, for an email marketing web app called WooshMail.

Problem to Solve

The admin user should have the highest level of access, while the basic should have access to a few features.

Target Audience

Admin User

  • Between the ages of 40 and 60.
  • Small business owners.
  • High School/GED or Bachelor's Degree.
  • Pursing their own passions.
  • Used their own funds to get started.

Basic User

  • Between the ages of 20 and 40.
  • Based in the United States.
  • Work for a small business.
  • High School/GED.

User Flow Testing

A total of 5 participants voluteered for the usability study. In summary, participants were instructed to navigate through the prototype and stop before sending an email. Moreover, participants completed the usability test twice, one for the admin user role and one for the basic user role.

User Flow Analysis

Most participants were unfamiliar with email marketing before they started the study. Thus, I believe offering a short video tutorial or description of how to use email marketing in the prototype would be helpful for users. In the basic prototype, I could have provided a little more detail on how to select and send an email given that some participants looked a little puzzled when viewing this section. Other changes to the prototype would follow participants' frustrations. For example, future prototypes will need to allow users revisit the editing page from the add recipients page and to properly label the send page, so there is no confusion who the user is sending the email to. Summary

In summary, I knew I had to design a simple and appealing website, because I felt that would set the tone for the entire site. As for the prototypes, I gained more confidence in conducting user research and learned to document participants' behavioral and facial expressions. Overall, I enjoyed working on this assignment and learned more about business-to-business design.
